## Decisions & Trade-offs
| Decision | Rationale |
|----------|-----------|
| createTerminal({env, cwd}) over sendText | Cross-platform: no shell syntax dependency |
| cwd separate from command | cd && is shell-specific; cwd API is universal |
| Auto-migrate cd && commands | Patch issue — upgrading users shouldn't hand-edit 15 terminals |
| subagentPermissionPending flag over string-concat label | Survives refreshFromConfig, configurable, clears correctly |
| Focus-loss only on waiting, not ready | Approve-and-switch-back is legitimate behavior |
| Single-read restore flow (no truncation) | Prevents RCE bypass via padded prefix or TOCTOU |
| O_EXCL for backup writes | Eliminates TOCTOU race in filename selection |
| Hook always writes optional fields | Prevents stale field leakage via Object.assign |
| Single global config (no workspace override) | One file, no merge logic, no confusion. Override deferred to #8 |
| Read-merge-write for concurrent windows | Handles two windows adding terminals simultaneously |
| 0.10/0.11 versioning, not 1.0 | Pre-1.0 until explicit user decision |
| QuickPick multi-step for wizard (Phase 3) | Native VS Code UX, testable per-step, cross-platform |
| Round-robin colors in wizard | Fast, no classification needed |
| Configurable scan depth, default 1 | Covers monorepo without full recursion |
| Ambiguous fuzzy match → skip | Better to drop update than guess wrong tile |
